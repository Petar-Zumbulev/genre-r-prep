# Gen Re Analyst Role Prep Roadmap

This roadmap is my structured preparation plan for a potential working
student analyst/coder role at Gen Re. The focus is not purely
theoretical actuarial work, but practical R-based analyst work: data
cleaning, reporting, visualization, dashboards, automation, and business
logic in an insurance context.

The main goal of this prep is to become interview-ready in R, build
strong job-relevant GitHub projects, and improve my ability to explain
code, reporting workflows, and insurance-related reasoning clearly.

## Main curriculum

| Day | Main topic | What I'm learning | Main output |
|----|----|----|----|
| Day 1 | Setup + R workflow foundations | R/RStudio setup, packages, scripts, Git/GitHub setup, basic project structure | Working R setup + repo connected |
| Day 2 | Tidyverse fundamentals | `filter()`, `select()`, `mutate()`, `arrange()`, `group_by()`, `summarise()` | First clean summary tables |
| Day 3 | Practice + writing code myself | Solving small wrangling tasks without hand-holding, strengthening syntax memory, Git push workflow | Drills completed and committed |
| Day 4 | ggplot + interpretation | Bar charts, line charts, reordering, `coord_flip()`, grouped plots, business interpretation | First analytical plots |
| Day 5 | Insurance reporting in R | Claim count, premium, severity, loss ratio, quarterly summaries, reusable reporting logic | Mini insurance report workflow |
| Day 6 | Shiny basics | App structure, `ui`, `server`, `shinyApp()`, simple outputs, simple inputs | First working Shiny app |
| Day 7 | Shiny dashboard logic | Filtering, reactive thinking, user controls, summary boxes, tables, plots together | Mini interactive dashboard |
| Day 8 | Excel workflows in R | `readxl`, `openxlsx`, importing Excel, cleaning sheets, exporting styled workbooks | Excel-to-R-to-Excel workflow |
| Day 9 | SQL / Postgres basics | `SELECT`, `WHERE`, `GROUP BY`, `JOIN`, basic analyst queries, linking SQL thinking to R | Small SQL practice set |
| Day 10 | APIs and JSON in R | API basics, GET requests, parsing JSON, turning nested data into tables | API-to-tibble workflow |
| Day 11 | PDF / OCR / unstructured data | Extracting text/tables from PDFs, understanding OCR use cases, cleaning messy inputs | Raw document to structured table |
| Day 12 | Insurance business logic | Severity, frequency, inflation effects, premium logic, segment comparisons | Business-metric notes + examples |
| Day 13 | Trend modeling basics | Smoothing, simple regression, explaining trends in claims/premiums over time | First simple modeling workflow |

## Project and interview phase

|----|----|----|----|
| Day 14 | Project 1 planning | Structuring the insurance reporting/dashboard project, folder layout, data flow, deliverables | Project skeleton |
| Day 15 | Project 1 build | Building the main R/Shiny insurance dashboard project | Core project working |

## Overall structure

| Phase | Days | Focus |
|----|----|----|
| Foundations | Days 1–4 | R basics, tidyverse, plotting |
| Insurance reporting | Day 5 | Practical claims and premium reporting |
| Tools | Days 6–11 | Shiny, Excel, SQL, APIs, PDF/OCR |
| Business logic + modeling | Days 12–13 | Insurance reasoning and simple trends |
| Portfolio + interview | Days 14–18 | Projects, GitHub polish, and mock prep |

## End goals

By the end of this roadmap, I want to have:

-   strong practical R workflow skills for analyst tasks
-   at least one strong insurance-related GitHub project
-   one smaller supporting workflow project
-   stronger confidence with reporting, dashboards, and data wrangling
-   clear interview readiness for coding tasks and business logic
    questions
